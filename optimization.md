SISS 2.0 Demo 優化項目清單
🔴 高優先（影響 Demo 體驗的核心問題）
1. 缺少登出功能
目前 TopBar 的使用者頭像是死連結，點了沒有任何反應。Demo 給客戶看時，若他們想重新展示登入流程，根本沒有登出按鈕。

解法：點擊 .topbar-user 時彈出下拉選單，包含「登出」選項，執行後返回登入頁並重置狀態
2. 通知鈴鐺無功能
TopBar 的 bell icon 有紅色 badge 顯示「3」筆通知，但點了完全沒有反應。客戶一定會去點它。

解法：點擊後彈出一個 dropdown 顯示 3 筆假通知訊息
3. 儀表板日期是硬編碼
page-dashboard 的 subtitle 寫死 2026年4月23日 星期四，這是死資料不是動態產生的。

解法：用 JS 動態產生當天日期
4. chart_monthly 每次重整都是亂數
data.js 用 Math.random() 產生折線圖資料，每次重新整理圖表數字都不同，Demo 時看起來不專業。

解法：換成固定的假資料陣列
🟡 中優先（Demo 加分項目）
5. Sidebar 收合時 tooltip 消失
Sidebar 收合後圖示還在，但 hover 時沒有顯示文字提示，使用者不知道每個 icon 是什麼功能。

解法：收合狀態下，nav-item hover 時顯示 CSS tooltip（title 屬性或 ::after 偽元素）
6. 清洗批次確認後 KPI 數字更新不完整
confirmWash() 只更新 kpi.wash - 1，但實際上完成清洗後「待包配盤包數」應該增加，邏輯上不連貫。

解法：同時 kpi.package += 1，讓數字連動更有說服力
7. 滅菌管理頁的「進行滅菌」按鈕對閒置/待檢驗狀態不應顯示
三台滅菌鍋都顯示「進行滅菌」按鈕，但「待檢驗」狀態的鍋應該顯示「進行檢驗」，「閒置中」顯示「開始使用」，語意不對。

解法：根據 s.status 條件渲染不同按鈕文字與行為
8. Placeholder 頁面缺乏引導感
目前只有「此功能模組開發中」一句話，略顯敷衍。業務 Demo 時客戶會問「這個頁面什麼時候好」。

解法：加上功能描述清單（列出該模組將提供的功能點），加上預計上線時程標示，視覺上更像「Coming Soon」而非「未完成」
9. 搜尋列按鈕與即時搜尋邏輯重複
#wash-search 的 oninput 已經即時呼叫 renderWashTable()，但又有個「搜尋」按鈕帶 loading。邏輯重複且混亂。

解法：統一為「只有按搜尋按鈕才觸發」，移除 oninput，體驗更乾淨
🟢 低優先（細節打磨）
10. Modal 的「確認完成清洗」對已完成批次隱藏按鈕，但沒有顯示狀態說明
已完成的批次打開 Modal，確認按鈕消失了，但沒有文字說明「此批次已完成清洗」，使用者會以為是 bug。

解法：按鈕消失時，在 footer 左側加一個綠色「已完成清洗」的 badge 說明
11. 儀表板 KPI 卡片可點擊但無動作
KPI 卡片有 hover 效果（transform: translateY(-2px)），暗示可點擊，但點了沒有任何動作。

解法：點擊對應 KPI 時跳轉到該功能頁面（如點「待清洗器械數」跳到清洗管理頁）
12. 沒有行動版（RWD）支援
CSS 在 <900px 只做了最基本的 grid 調整，但 sidebar 在手機上會擠壓內容，Demo 若在平板上展示會跑版。

解法：<768px 時讓 sidebar 變成 overlay 模式（fixed position + 背景遮罩）
